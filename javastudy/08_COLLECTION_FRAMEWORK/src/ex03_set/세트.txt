Set 인터페이스

1. 수학의 집합을 관리하는 자료구조이다.
2. 특징 (특징 두 가지가 중요하다)
	1) 저장된 요소 간의 순서가 없다. (인덱스가 없다)  중요!
	2) 중복된 요소 저장이 불가능하다. (unique하다)    중요!
3. HashSet 클래스
	1) 해시 기반의 Set 처리 클래스이다.
	2) 해시(Hash) : 특정 인스턴스가 가지는 값. 빠른 검색을 위해서 사용
4. HashSet 클래스의 중복 요소 여부 판단
	1) 2가지 메소드를 사용한다.
	2) hashCode() 메소드를 호출해서 해시(Hash)값을 확인한다.
		- 동일한 해시 : 중복된 요소일 가능성이 있다. 추가로 equals()비교를 한다.
		- 다른 해시 : 중복된 요소가 아님 확정. 더 이상 비교 안함.
	3) equals() 메소드를 호출해서 동등 비교를 수행함.
	4) 실무에서는 hashCode(), equals() 메소드를 동시에 자동 생성함.
	
	
	
	탐색 방식
	1) full scan : 하나씩 끝까지 다 탐색 (성능이 가장 안좋다)
	2) 해시 기반 : 해시 값을 기반으로 데이터를 분류해놓고, 검색할 데이터의 해시 값에 해당하는 데이터만 탐색하는 방법
